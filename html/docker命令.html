-i：保持标准输入保持打开状态，一般与 -t 连用
-t：分配一个伪终端，一般与 -i 连用
-d：在后台运行容器，并且打印容器id
常规操作：
1.docker引擎启动：systemctl start docker
2.拉取镜像：docker pull 镜像名
3.查看镜像：docker images
4.删除镜像：docker rmi -f 镜像名
5.创建容器：docker run -it -d --name 创建的容器名 镜像名
6.查看所有容器：docker ps -a
7.以交互式方式启动容器：docker run -it 镜像名 /bin/bash
8.进入容器：docker attach 容器id
